Roadmap

The most important next step for Jaql is to open-source the code and find a home where we can streamline our interaction with the community. Following this important milestone, there is no shortage of interesting problems left to solve in Jaql, and we welcome help on it. These include:

  • Enable bindings to other programming languages besides Java so Jaql clients and functions can be written in, say, Python or JavaScript.
  • Upgrade the version of Hadoop that Jaql depends on.
  • Investigate how Facebook's Thrift could be leveraged by Jaql.
  • Performance testing and performance improvements.
  • Extend the notion of job tracking, load balancing, and restarting to Jaql queries.
  • Investigate how schema may be specified for JSON data and how it can be exploited for storage and query processing.
  • Support more recent versions of Hadoop and HBase.
